# 1. 设计目标

上一版 `RankingService` 的问题是：

- 主要按 `confidence` 排
- 对“根因规则”和“症状规则”区分不够强
- 对同一类问题的重复命中没有去重
- 对越上游越优先的原则体现不够

增强版的目标是：

### 目标 1：根因优先
即使某条症状规则 confidence 不低，也不能轻易压过真正根因规则。

### 目标 2：上游优先
链路越上游，越可能是真根因，应优先于下游表象。

### 目标 3：cluster 去重
同一类根因不要霸榜，例如：

- `R201 合法 falsy 被吞`
- `R202 formatter 输出 fallback`
- `R501 页面显示 fallback`

这三条本质上可能是同一个问题簇。

### 目标 4：症状降权
例如：
- 页面显示 `--`
- DOM 看起来没更新

这些都应该更像 symptom，而不是 top cause。

# 2. 增强版排序策略

---

## 2.1 最终分数公式

```ts
finalScore =
  baseScore
  + rootCauseScore
  + layerPriorityScore
  + specificityScore
  + evidenceScore
  + chainConsistencyScore
  - symptomPenalty
  - duplicatePenalty
```

---

## 2.2 各项含义

### baseScore
来自 `confidence * 100`

### rootCauseScore
如果不是 symptom，则额外加分

### layerPriorityScore
越上游加分越高：

- `api` / `api_to_state` / `state` / `selector` / `render`：高
- `dom`：中
- `ui`：低

### specificityScore
规则越具体，分越高。  
例如 cluster 存在，且不是泛化症状规则，加分更高。

### evidenceScore
证据 refs 越完整，加分越高。

### chainConsistencyScore
上下游证据越连续，加分越高。

### symptomPenalty
症状规则降权。

### duplicatePenalty
同 cluster 的规则，只有最强的一条保留为主候选，其余降权。

```ts
import { Injectable } from '@nestjs/common';
import { DiagnosisContext } from '../models/diagnosis-context.model';
import {
  EnrichedFinding,
  RankScore,
  RuleFinding,
} from '../models/rule-finding.model';

@Injectable()
export class RankingService {
  rank(findings: RuleFinding[], context: DiagnosisContext): EnrichedFinding[] {
    const enriched = findings.map((finding) => {
      const score = this.calculateScore(finding, context);
      return {
        ...finding,
        rankScore: score,
        rankReasons: this.buildReasons(finding, context, score),
      };
    });

    const deduped = this.applyClusterDedup(enriched);

    return deduped.sort(
      (a, b) => b.rankScore.finalScore - a.rankScore.finalScore,
    );
  }

  private calculateScore(
    finding: RuleFinding,
    context: DiagnosisContext,
  ): RankScore {
    const baseScore = Math.round((finding.confidence ?? 0) * 100);

    const rootCauseScore = this.getRootCauseScore(finding);

    const layerPriorityScore = this.getLayerPriorityScore(finding.layer);

    const specificityScore = this.getSpecificityScore(finding);

    const evidenceScore = this.getEvidenceScore(finding);

    const chainConsistencyScore = this.getChainConsistencyScore(
      finding,
      context,
    );

    const symptomPenalty = this.getSymptomPenalty(finding);

    const duplicatePenalty = 0;

    const finalScore =
      baseScore +
      rootCauseScore +
      layerPriorityScore +
      specificityScore +
      evidenceScore +
      chainConsistencyScore -
      symptomPenalty -
      duplicatePenalty;

    return {
      ruleCode: finding.ruleCode,
      baseScore,
      rootCauseScore,
      specificityScore,
      evidenceScore,
      chainConsistencyScore,
      symptomPenalty,
      duplicatePenalty,
      finalScore,
    };
  }

  private applyClusterDedup(
    findings: EnrichedFinding[],
  ): EnrichedFinding[] {
    const sorted = [...findings].sort(
      (a, b) => b.rankScore.finalScore - a.rankScore.finalScore,
    );

    const clusterWinnerMap = new Map<string, EnrichedFinding>();
    const result: EnrichedFinding[] = [];

    for (const item of sorted) {
      const clusterKey = item.cluster?.trim();

      if (!clusterKey) {
        result.push(item);
        continue;
      }

      const existing = clusterWinnerMap.get(clusterKey);

      if (!existing) {
        clusterWinnerMap.set(clusterKey, item);
        result.push(item);
        continue;
      }

      const penalized = this.withDuplicatePenalty(item, existing);
      result.push(penalized);
    }

    return result;
  }

  private withDuplicatePenalty(
    current: EnrichedFinding,
    winner: EnrichedFinding,
  ): EnrichedFinding {
    const isSameCluster = current.cluster && current.cluster === winner.cluster;

    if (!isSameCluster) return current;

    const penalty = this.getDuplicatePenalty(current, winner);

    const finalScore = current.rankScore.finalScore - penalty;

    return {
      ...current,
      rankScore: {
        ...current.rankScore,
        duplicatePenalty: penalty,
        finalScore,
      },
      rankReasons: [
        ...(current.rankReasons ?? []),
        `与同 cluster 规则 ${winner.ruleCode} 重叠，已降权`,
      ],
    };
  }

  private getRootCauseScore(finding: RuleFinding): number {
    if (finding.isSymptomOnly) return 0;

    const category = finding.category;
    const layer = finding.layer;

    if (
      category === 'data_source' ||
      category === 'state_binding' ||
      layer === 'api' ||
      layer === 'api_to_state' ||
      layer === 'state' ||
      layer === 'selector' ||
      layer === 'render'
    ) {
      return 22;
    }

    if (category === 'interaction' || layer === 'handler') {
      return 18;
    }

    if (category === 'dom') {
      return 10;
    }

    return 8;
  }

  private getLayerPriorityScore(layer?: string): number {
    switch (layer) {
      case 'api':
        return 18;
      case 'api_to_state':
        return 16;
      case 'state':
        return 15;
      case 'selector':
        return 14;
      case 'render':
        return 13;
      case 'handler':
        return 12;
      case 'request_to_ui':
        return 10;
      case 'dom':
        return 7;
      case 'ui':
        return 4;
      default:
        return 5;
    }
  }

  private getSpecificityScore(finding: RuleFinding): number {
    let score = 0;

    if (finding.cluster) score += 10;

    if (!finding.isSymptomOnly) score += 6;

    if (finding.ruleCode.startsWith('R1')) score += 6;
    else if (finding.ruleCode.startsWith('R2')) score += 8;
    else if (finding.ruleCode.startsWith('R3')) score += 5;
    else if (finding.ruleCode.startsWith('R4')) score += 6;
    else if (finding.ruleCode.startsWith('R5')) score += 2;

    if ((finding.evidenceRefs?.length ?? 0) >= 2) score += 4;

    return Math.min(score, 24);
  }

  private getEvidenceScore(finding: RuleFinding): number {
    const count = finding.evidenceRefs?.length ?? 0;

    if (count >= 4) return 18;
    if (count === 3) return 15;
    if (count === 2) return 12;
    if (count === 1) return 7;
    return 0;
  }

  private getChainConsistencyScore(
    finding: RuleFinding,
    context: DiagnosisContext,
  ): number {
    let score = 0;

    const hasApiEvidence =
      context.responseSuccess === true || context.apiValue !== undefined;
    const hasStateEvidence =
      context.storeUpdated === true || context.storeValue !== undefined;
    const hasRenderEvidence =
      context.renderTriggered === true || context.renderOutputValue !== undefined;
    const hasDomEvidence =
      context.domUpdated === true || context.domValue !== undefined;

    if (
      finding.layer === 'api' ||
      finding.layer === 'api_to_state' ||
      finding.layer === 'state' ||
      finding.layer === 'selector'
    ) {
      if (hasApiEvidence) score += 4;
      if (hasStateEvidence) score += 4;
      if (hasRenderEvidence) score += 2;
    } else if (finding.layer === 'render') {
      if (hasStateEvidence) score += 3;
      if (hasRenderEvidence) score += 5;
      if (hasDomEvidence) score += 2;
    } else if (finding.layer === 'dom') {
      if (hasRenderEvidence) score += 4;
      if (hasDomEvidence) score += 4;
    } else if (finding.layer === 'handler' || finding.layer === 'request_to_ui') {
      if (context.clickDetected === true) score += 3;
      if (context.handlerStarted === true) score += 3;
      if (context.requestSent !== undefined) score += 2;
    } else {
      if (hasApiEvidence || hasStateEvidence || hasRenderEvidence || hasDomEvidence) {
        score += 4;
      }
    }

    return Math.min(score, 12);
  }

  private getSymptomPenalty(finding: RuleFinding): number {
    if (!finding.isSymptomOnly) return 0;

    if (finding.ruleCode === 'R501') return 20;
    if (finding.ruleCode === 'R502') return 18;
    if (finding.ruleCode === 'R503') return 18;
    if (finding.ruleCode === 'R504') return 24;

    return 16;
  }

  private getDuplicatePenalty(
    current: EnrichedFinding,
    winner: EnrichedFinding,
  ): number {
    if (current.ruleCode === winner.ruleCode) return 30;

    let penalty = 14;

    if (current.isSymptomOnly) penalty += 6;

    if (current.layer === winner.layer) penalty += 4;

    if (current.category === winner.category) penalty += 3;

    if (this.isMoreGenericThan(current, winner)) penalty += 8;

    return penalty;
  }

  private isMoreGenericThan(
    current: EnrichedFinding,
    winner: EnrichedFinding,
  ): boolean {
    const genericRules = new Set(['R202', 'R203', 'R501', 'R502', 'R503', 'R504']);
    const specificRules = new Set(['R101', 'R102', 'R103', 'R104', 'R201', 'R301', 'R302', 'R401', 'R402']);

    if (genericRules.has(current.ruleCode) && specificRules.has(winner.ruleCode)) {
      return true;
    }

    if (current.isSymptomOnly && !winner.isSymptomOnly) {
      return true;
    }

    return false;
  }

  private buildReasons(
    finding: RuleFinding,
    context: DiagnosisContext,
    score: RankScore,
  ): string[] {
    const reasons: string[] = [];

    if (!finding.isSymptomOnly) {
      reasons.push('根因规则优先');
    } else {
      reasons.push('症状规则已降权');
    }

    const layerReason = this.getLayerReason(finding.layer);
    if (layerReason) reasons.push(layerReason);

    if (finding.cluster) {
      reasons.push('规则簇明确，可归入稳定问题类型');
    }

    const evidenceCount = finding.evidenceRefs?.length ?? 0;
    if (evidenceCount >= 3) {
      reasons.push('证据链完整度高');
    } else if (evidenceCount >= 2) {
      reasons.push('证据链较完整');
    } else if (evidenceCount >= 1) {
      reasons.push('存在直接证据');
    }

    if (score.baseScore >= 90) {
      reasons.push('规则置信度高');
    } else if (score.baseScore >= 80) {
      reasons.push('规则置信度较高');
    }

    if (score.chainConsistencyScore >= 8) {
      reasons.push('上下游链路证据一致');
    }

    if (
      context.apiValue !== undefined &&
      context.renderOutputValue !== undefined &&
      finding.layer === 'render'
    ) {
      reasons.push('上游输入与渲染输出可直接对照');
    }

    return reasons;
  }

  private getLayerReason(layer?: string): string | null {
    switch (layer) {
      case 'api':
        return '上游 API 层优先级最高';
      case 'api_to_state':
        return 'API 到状态映射层属于高优先级根因位置';
      case 'state':
        return '状态写入层属于高优先级根因位置';
      case 'selector':
        return 'selector 层属于高优先级数据绑定断点';
      case 'render':
        return 'render 层属于核心转换断点';
      case 'handler':
        return '交互处理层是点击链路关键断点';
      case 'request_to_ui':
        return '请求前链路属于交互诊断关键位置';
      case 'dom':
        return 'DOM 层更偏结果承接与呈现';
      case 'ui':
        return 'UI 层规则更偏表象';
      default:
        return null;
    }
  }
}
```

# 4. 这版排序逻辑的关键点

---

## 4.1 根因优先
通过 `getRootCauseScore()` 明确拉开：

- 非 symptom 规则：高分
- symptom 规则：不加分

特别是：

- `data_source`
- `state_binding`
- `render`
- `selector`

这些都天然更接近根因。

---

## 4.2 上游优先
通过 `getLayerPriorityScore()` 体现链路位置权重：

```ts
api > api_to_state > state > selector > render > handler > request_to_ui > dom > ui
```

这意味着：

- `R201` 往往会压过 `R501`
- `R103` 往往会压过 `R302`
- `R101/R102` 往往会压过 DOM 层症状

---

## 4.3 cluster 去重
`applyClusterDedup()` 的逻辑是：

- 每个 cluster 先选一个 winner
- 后续同 cluster 规则继续保留，但打 `duplicatePenalty`
- 不会直接删除，方便后面作为 supporting / symptom 展示

这样做有两个好处：

### 好处 1：保留信息
不会因为去重把有价值的 supporting cause 直接丢掉。

### 好处 2：避免霸榜
同一类问题不会把 Top 3 都占满。

---

## 4.4 症状降权
`getSymptomPenalty()` 对 `R501-R504` 做了显式减分：

- `R504` 罚得最重，因为它只是“证据不足”
- `R501-R503` 是中等症状惩罚

这样保证 symptom 不会轻易抢到 top cause。

---

## 4.5 泛化规则进一步降权
`isMoreGenericThan()` 里把这些规则视为更泛的规则：

```ts
R202, R203, R501, R502, R503, R504
```

如果它们和更具体规则同 cluster，就会被额外惩罚。

例如：

- `R201` vs `R202`
- `R201` vs `R501`

在相同问题簇里，`R201` 应该胜出。

# 5. 与 DiagnosisConclusionService 的衔接

你前面的 `DiagnosisConclusionService` 基本不用改，只要继续这样调用：

```ts
const ranked = this.rankingService.rank(findings, context);
const topFinding = ranked[0] ?? null;

const supportingFindings = ranked.filter(
  (item, index) =>
    index > 0 &&
    !item.isSymptomOnly &&
    item.cluster !== topFinding?.cluster,
);

const symptomFindings = ranked.filter((item) => item.isSymptomOnly);
```

如果你想让“同 cluster 的重复根因”直接不进 supporting，也可以再加一道过滤：

```ts
const usedClusters = new Set<string>();
const supportingFindings = ranked.filter((item, index) => {
  if (index === 0) return false;
  if (item.isSymptomOnly) return false;

  const cluster = item.cluster ?? `__no_cluster__:${item.ruleCode}`;
  if (usedClusters.has(cluster)) return false;
  usedClusters.add(cluster);

  return cluster !== topFinding?.cluster;
});
```

不过 MVP 阶段其实你现有逻辑已经够用了。

# 6. 一个实际效果示例

假设同时命中：

- `R201` 合法 falsy 被吞
- `R202` formatter 输出 fallback
- `R501` 页面显示 fallback 占位值

这版排序大概率会变成：

1. `R201`
2. `R202`
3. `R501`

原因：

- `R201` 更具体
- `R201` 是根因规则
- `R202` 更泛
- `R501` 是症状规则，显式降权

这正是你想要的“根因优先 + 症状兜底”。

# 7. 这版比上一版提升了什么

相比你前一个版本，这版主要提升在：

### 提升 1：显式 layer 权重
不是只有 confidence，而是体现链路断点位置价值。

### 提升 2：同 cluster 不再平铺竞争
避免同一问题簇内多个规则挤占前列。

### 提升 3：症状规则不会抢根因
工作台结果会更稳定。

### 提升 4：rank reasons 更可解释
后面可以直接展示给用户，例如：

- 根因规则优先
- 上游 API 层优先级最高
- 证据链完整度高
- 上下游链路证据一致

这对 Diagnosis 结果页很有用。
