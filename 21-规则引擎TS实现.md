# 1. 规则范围与编码约定

这里先给一版 **P0 / P1**，覆盖 MVP 最有价值的链路断点：

---

## P0：高优先级根因规则

### Data Source
- `R101` 接口返回成功但字段缺失
- `R102` 接口字段路径配置错误 / 绑定路径失效

### State Binding
- `R103` 请求成功但未触发状态更新
- `R104` 状态已更新但 selector 未取到值

### Render Transform
- `R201` 合法 falsy 值被误判为空
- `R202` formatter / transform 输出 fallback 占位值
- `R203` render 输入正确但输出与预期不一致

### DOM / UI
- `R301` DOM 已渲染但被隐藏
- `R302` DOM 未更新，页面仍显示旧值/占位值

### Interaction
- `R401` click 已发生但 handler 未启动
- `R402` handler 已执行但 request 未发出

---

## P1：症状 / 辅助规则

- `R501` 页面显示 fallback 占位值
- `R502` 链路中断于 render 之后 DOM 之前
- `R503` 链路中断于 request 之前
- `R504` 证据不足但存在异常表象

# 3. 完整代码

```ts
import { Injectable } from '@nestjs/common';
import { DiagnosisContext } from '../models/diagnosis-context.model';
import { RuleFinding } from '../models/rule-finding.model';

@Injectable()
export class RuleEngineService {
  run(context: DiagnosisContext): RuleFinding[] {
    const findings: RuleFinding[] = [];

    // P0 - Data Source
    this.pushIfHit(findings, this.hitApiFieldMissing(context)); // R101
    this.pushIfHit(findings, this.hitApiBindingPathInvalid(context)); // R102

    // P0 - State Binding
    this.pushIfHit(findings, this.hitRequestSuccessButStateNotUpdated(context)); // R103
    this.pushIfHit(findings, this.hitStateUpdatedButSelectorMissed(context)); // R104

    // P0 - Render Transform
    this.pushIfHit(findings, this.hitFalsyValueSwallowed(context)); // R201
    this.pushIfHit(findings, this.hitFormatterReturnedFallback(context)); // R202
    this.pushIfHit(findings, this.hitRenderOutputUnexpected(context)); // R203

    // P0 - DOM / UI
    this.pushIfHit(findings, this.hitDomRenderedButHidden(context)); // R301
    this.pushIfHit(findings, this.hitDomNotUpdated(context)); // R302

    // P0 - Interaction
    this.pushIfHit(findings, this.hitClickButHandlerNotStarted(context)); // R401
    this.pushIfHit(findings, this.hitHandlerStartedButRequestNotSent(context)); // R402

    // P1 - Symptom / Assist
    this.pushIfHit(findings, this.hitFallbackDisplayedSymptom(context)); // R501
    this.pushIfHit(findings, this.hitRenderDomGapSymptom(context)); // R502
    this.pushIfHit(findings, this.hitPreRequestGapSymptom(context)); // R503
    this.pushIfHit(findings, this.hitInsufficientEvidenceSymptom(context)); // R504

    return this.deduplicate(findings);
  }

  private pushIfHit(
    findings: RuleFinding[],
    finding: RuleFinding | null,
  ): void {
    if (finding) findings.push(finding);
  }

  private deduplicate(findings: RuleFinding[]): RuleFinding[] {
    const seen = new Set<string>();
    const result: RuleFinding[] = [];

    for (const item of findings) {
      const key = `${item.ruleCode}:${item.cluster ?? ''}:${item.layer}`;
      if (seen.has(key)) continue;
      seen.add(key);
      result.push(item);
    }

    return result;
  }

  // =========================================================
  // Data Source Rules
  // =========================================================

  /**
   * R101: 接口返回成功但字段缺失
   * 条件：
   * - responseSuccess = true
   * - apiFieldPath 存在
   * - apiFieldExists = false
   */
  private hitApiFieldMissing(context: DiagnosisContext): RuleFinding | null {
    if (context.responseSuccess !== true) return null;
    if (!context.apiFieldPath) return null;
    if (context.apiFieldExists !== false) return null;

    return {
      ruleCode: 'R101',
      title: '接口返回成功但字段缺失',
      diagnosisLabel: '接口字段缺失',
      category: 'data_source',
      severity: 'high',
      confidence: 0.95,
      layer: 'api',
      cluster: 'api_field_missing',
      summary: `接口请求成功，但目标字段 ${context.apiFieldPath} 未出现在返回结果中。`,
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_api_field_missing',
        'ev_api_amount_0',
      ]),
      suggestions: [
        '检查接口 schema 是否变更',
        '检查目标字段路径配置是否过期',
        '确认后端返回结构与前端绑定路径一致',
      ],
      isSymptomOnly: false,
      metadata: {
        apiFieldPath: context.apiFieldPath,
      },
    };
  }

  /**
   * R102: 接口字段路径配置错误 / 绑定路径失效
   * 条件（启发式）：
   * - responseSuccess = true
   * - apiFieldPath 存在
   * - apiFieldExists = false
   * - 同时 storeKey / selectorName / formatterName 中至少一个存在
   * 说明：
   * - 它和 R101 有重叠，但 R101 更偏“后端没给”
   * - R102 更偏“前端绑定路径错了”
   * 当前先保留两者，交给 ranking/conclusion 做排序
   */
  private hitApiBindingPathInvalid(
    context: DiagnosisContext,
  ): RuleFinding | null {
    if (context.responseSuccess !== true) return null;
    if (!context.apiFieldPath) return null;
    if (context.apiFieldExists !== false) return null;

    const hasDownstreamBinding =
      Boolean(context.storeKey) ||
      Boolean(context.selectorName) ||
      Boolean(context.formatterName);

    if (!hasDownstreamBinding) return null;

    return {
      ruleCode: 'R102',
      title: '接口字段路径配置错误或绑定失效',
      diagnosisLabel: '接口绑定路径失效',
      category: 'data_source',
      severity: 'high',
      confidence: 0.88,
      layer: 'api_to_state',
      cluster: 'api_binding_path_invalid',
      summary: `接口成功返回，但绑定路径 ${context.apiFieldPath} 未取到值，疑似字段路径配置错误或 schema 映射失效。`,
      evidenceRefs: this.pickEvidenceRefs(context, ['ev_api_field_missing']),
      suggestions: [
        '核对 API 字段路径配置',
        '检查数据映射层是否仍按旧 schema 取值',
        '对绑定路径增加 schema 变更告警',
      ],
      isSymptomOnly: false,
      metadata: {
        apiFieldPath: context.apiFieldPath,
        storeKey: context.storeKey,
      },
    };
  }

  // =========================================================
  // State Binding Rules
  // =========================================================

  /**
   * R103: 请求成功但未触发状态更新
   * 条件：
   * - requestSent = true
   * - responseSuccess = true
   * - storeUpdated = false
   */
  private hitRequestSuccessButStateNotUpdated(
    context: DiagnosisContext,
  ): RuleFinding | null {
    if (context.requestSent !== true) return null;
    if (context.responseSuccess !== true) return null;
    if (context.storeUpdated !== false) return null;

    return {
      ruleCode: 'R103',
      title: '请求成功但未触发状态更新',
      diagnosisLabel: '请求成功但状态未更新',
      category: 'state_binding',
      severity: 'high',
      confidence: 0.93,
      layer: 'state',
      cluster: 'request_success_state_not_updated',
      summary: '接口请求已成功返回，但 store 未发生对应更新，链路中断在数据写入状态层之前。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_api_amount_0',
        'ev_store_update_missing',
      ]),
      suggestions: [
        '检查 request success 回调是否 dispatch/commit 状态更新',
        '检查 reducer / mutation / action 是否执行',
        '检查状态写入条件是否被提前 return',
      ],
      isSymptomOnly: false,
      metadata: {
        storeKey: context.storeKey,
      },
    };
  }

  /**
   * R104: 状态已更新但 selector 未取到值
   * 条件：
   * - storeUpdated = true
   * - selectorRan = true 或 selectorName 存在
   * - selectorValue = undefined/null
   */
  private hitStateUpdatedButSelectorMissed(
    context: DiagnosisContext,
  ): RuleFinding | null {
    if (context.storeUpdated !== true) return null;

    const hasSelectorStage =
      context.selectorRan === true || Boolean(context.selectorName);
    if (!hasSelectorStage) return null;

    if (!this.isNil(context.selectorValue)) return null;

    return {
      ruleCode: 'R104',
      title: '状态已更新但 selector 未取到值',
      diagnosisLabel: 'selector 取值失败',
      category: 'state_binding',
      severity: 'high',
      confidence: 0.91,
      layer: 'selector',
      cluster: 'selector_value_missing',
      summary: '状态层已经更新，但 selector 未能取到目标值，疑似 selector 路径错误、依赖断裂或 memo 命中异常。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_store_value',
        'ev_selector_value_missing',
      ]),
      suggestions: [
        '检查 selector 输入路径是否正确',
        '检查 selector 依赖字段是否变更',
        '检查 memo/cache 条件是否错误命中旧结果',
      ],
      isSymptomOnly: false,
      metadata: {
        storeKey: context.storeKey,
        selectorName: context.selectorName,
      },
    };
  }

  // =========================================================
  // Render Transform Rules
  // =========================================================

  /**
   * R201: 合法 falsy 值被误判为空
   * 条件：
   * - 上游某层值为 0 / false
   * - renderOutputValue 看起来是 fallback
   */
  private hitFalsyValueSwallowed(
    context: DiagnosisContext,
  ): RuleFinding | null {
    const hasLegalFalsyValue =
      context.apiValue === 0 ||
      context.storeValue === 0 ||
      context.selectorValue === 0 ||
      context.renderInputValue === 0 ||
      context.apiValue === false ||
      context.storeValue === false ||
      context.selectorValue === false ||
      context.renderInputValue === false;

    if (!hasLegalFalsyValue) return null;

    const outputLooksFallback =
      context.formatterOutputIsFallback === true ||
      this.isFallbackValue(context.renderOutputValue, context.fallbackTokens) ||
      this.isFallbackValue(context.domValue, context.fallbackTokens);

    if (!outputLooksFallback) return null;

    return {
      ruleCode: 'R201',
      title: '合法 falsy 值被误判为空',
      diagnosisLabel: '合法值被判空吞掉',
      category: 'render_transform',
      severity: 'high',
      confidence: 0.96,
      layer: 'render',
      cluster: 'formatter_falsy_swallowed',
      summary:
        '上游存在合法值 0/false，但渲染或格式化逻辑将其按空值处理，最终输出占位值。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_api_amount_0',
        'ev_store_value',
        'ev_formatter_output_fallback',
      ]),
      suggestions: [
        '将 if (!value) 改为 value == null',
        '对 0、false、空字符串分别处理，避免与 null/undefined 混淆',
        '为 formatter 增加 falsy 边界值测试',
      ],
      isSymptomOnly: false,
      metadata: {
        renderInputValue: context.renderInputValue,
        renderOutputValue: context.renderOutputValue,
      },
    };
  }

  /**
   * R202: formatter / transform 输出 fallback 占位值
   * 条件：
   * - renderTriggered = true
   * - renderOutputValue 是 fallback
   * - renderInputValue 不是 null/undefined
   *
   * 说明：
   * - 它比 R201 更泛化
   * - R201 是“合法 falsy 被吞”
   * - R202 是“transform 输出了 fallback”
   */
  private hitFormatterReturnedFallback(
    context: DiagnosisContext,
  ): RuleFinding | null {
    if (context.renderTriggered !== true) return null;
    if (this.isNil(context.renderInputValue)) return null;

    const outputLooksFallback =
      context.formatterOutputIsFallback === true ||
      this.isFallbackValue(context.renderOutputValue, context.fallbackTokens);

    if (!outputLooksFallback) return null;

    return {
      ruleCode: 'R202',
      title: '格式化输出为 fallback 占位值',
      diagnosisLabel: 'formatter 输出占位值',
      category: 'render_transform',
      severity: 'high',
      confidence: 0.89,
      layer: 'render',
      cluster: 'formatter_fallback',
      summary: 'render 已执行且拿到输入值，但格式化/转换结果变成了占位值，疑似 transform 逻辑异常。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_formatter_output_fallback',
      ]),
      suggestions: [
        '检查 formatter/transform 的空值兜底逻辑',
        '检查输入值类型与 formatter 预期是否一致',
        '为 formatter 增加输入/输出 trace',
      ],
      isSymptomOnly: false,
      metadata: {
        formatterName: context.formatterName,
        renderInputValue: context.renderInputValue,
        renderOutputValue: context.renderOutputValue,
      },
    };
  }

  /**
   * R203: render 输入正确但输出与预期不一致
   * 条件（启发式）：
   * - renderTriggered = true
   * - renderInputValue 有值
   * - renderOutputValue 有值
   * - 输入输出不相等，且输出不是 DOM 最终值或明显异常
   *
   * 这个规则比较泛，作为 P1.5 / P0 辅助根因。
   */
  private hitRenderOutputUnexpected(
    context: DiagnosisContext,
  ): RuleFinding | null {
    if (context.renderTriggered !== true) return null;
    if (this.isNil(context.renderInputValue)) return null;
    if (this.isNil(context.renderOutputValue)) return null;

    const sameAsInput = this.looseEqual(
      context.renderInputValue,
      context.renderOutputValue,
    );

    if (sameAsInput) return null;

    const outputLooksFallback = this.isFallbackValue(
      context.renderOutputValue,
      context.fallbackTokens,
    );

    if (outputLooksFallback) return null; // 更具体的 R202 / R201 会接住

    return {
      ruleCode: 'R203',
      title: '渲染输出与输入不一致',
      diagnosisLabel: 'render 输出异常',
      category: 'render_transform',
      severity: 'medium',
      confidence: 0.78,
      layer: 'render',
      cluster: 'render_output_unexpected',
      summary: 'render/transform 拿到了输入值，但输出结果与输入值不一致，可能存在转换逻辑、格式化规则或类型处理异常。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_render_trace',
        'ev_formatter_output_fallback',
      ]),
      suggestions: [
        '检查 render transform/formatter 的输出逻辑',
        '对输入输出增加类型断言和日志',
        '补充 render 层单元测试',
      ],
      isSymptomOnly: false,
      metadata: {
        renderInputValue: context.renderInputValue,
        renderOutputValue: context.renderOutputValue,
      },
    };
  }

  // =========================================================
  // DOM / UI Rules
  // =========================================================

  /**
   * R301: DOM 已渲染但被隐藏
   * 条件：
   * - domUpdated = true
   * - domVisible = false
   */
  private hitDomRenderedButHidden(
    context: DiagnosisContext,
  ): RuleFinding | null {
    if (context.domUpdated !== true) return null;
    if (context.domVisible !== false) return null;

    return {
      ruleCode: 'R301',
      title: 'DOM 已渲染但被隐藏',
      diagnosisLabel: 'DOM 被隐藏',
      category: 'dom',
      severity: 'high',
      confidence: 0.9,
      layer: 'dom',
      cluster: 'dom_hidden',
      summary: '目标 DOM 已发生更新，但元素不可见，疑似被条件渲染、display:none、visibility:hidden 或层级遮挡影响。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_dom_snapshot',
        'ev_computed_style_hidden',
      ]),
      suggestions: [
        '检查 v-if/ngIf/条件分支是否隐藏元素',
        '检查 display/visibility/opacity/z-index 等样式',
        '检查父容器裁剪或遮挡问题',
      ],
      isSymptomOnly: false,
      metadata: {
        domValue: context.domValue,
      },
    };
  }

  /**
   * R302: DOM 未更新，页面仍显示旧值/占位值
   * 条件：
   * - renderTriggered = true
   * - domUpdated = false
   * 或
   * - renderOutputValue 与 domValue 不一致
   */
  private hitDomNotUpdated(context: DiagnosisContext): RuleFinding | null {
    const renderRan = context.renderTriggered === true;
    const domNotUpdated = context.domUpdated === false;

    const renderDomMismatch =
      !this.isNil(context.renderOutputValue) &&
      !this.isNil(context.domValue) &&
      !this.looseEqual(context.renderOutputValue, context.domValue);

    if (!renderRan && !renderDomMismatch) return null;
    if (!domNotUpdated && !renderDomMismatch) return null;

    return {
      ruleCode: 'R302',
      title: 'DOM 未更新或显示值与渲染输出不一致',
      diagnosisLabel: 'DOM 未反映最新渲染结果',
      category: 'dom',
      severity: 'high',
      confidence: renderDomMismatch ? 0.88 : 0.84,
      layer: 'dom',
      cluster: 'dom_not_updated',
      summary: '渲染层已经产出结果，但 DOM 未同步更新，或 DOM 显示值与 render 输出不一致。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_render_trace',
        'ev_dom_snapshot',
      ]),
      suggestions: [
        '检查组件是否实际 commit/render 到 DOM',
        '检查 shouldUpdate / memo / PureComponent 等优化是否阻止更新',
        '检查异步更新时机与目标节点绑定是否正确',
      ],
      isSymptomOnly: false,
      metadata: {
        renderOutputValue: context.renderOutputValue,
        domValue: context.domValue,
      },
    };
  }

  // =========================================================
  // Interaction Rules
  // =========================================================

  /**
   * R401: click 已发生但 handler 未启动
   * 条件：
   * - mode = click_diagnosis
   * - clickDetected = true
   * - handlerStarted = false
   */
  private hitClickButHandlerNotStarted(
    context: DiagnosisContext,
  ): RuleFinding | null {
    if (context.mode !== 'click_diagnosis') return null;
    if (context.clickDetected !== true) return null;
    if (context.handlerStarted !== false) return null;

    return {
      ruleCode: 'R401',
      title: '点击已发生但事件处理未启动',
      diagnosisLabel: 'click 未触发 handler',
      category: 'interaction',
      severity: 'high',
      confidence: 0.94,
      layer: 'handler',
      cluster: 'click_handler_not_started',
      summary: '点击事件已被捕获，但目标 handler 未执行，疑似事件绑定丢失、冒泡被阻断或 disabled 状态导致。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_click_trace',
        'ev_handler_missing',
      ]),
      suggestions: [
        '检查事件绑定是否丢失',
        '检查 disabled / pointer-events / 冒泡阻断',
        '检查虚拟节点替换后旧 handler 失效问题',
      ],
      isSymptomOnly: false,
      metadata: {},
    };
  }

  /**
   * R402: handler 已执行但 request 未发出
   * 条件：
   * - handlerStarted = true
   * - requestSent = false
   */
  private hitHandlerStartedButRequestNotSent(
    context: DiagnosisContext,
  ): RuleFinding | null {
    if (context.handlerStarted !== true) return null;
    if (context.requestSent !== false) return null;

    return {
      ruleCode: 'R402',
      title: '事件处理已执行但请求未发出',
      diagnosisLabel: 'handler 执行但 request 未发送',
      category: 'interaction',
      severity: 'high',
      confidence: 0.92,
      layer: 'request_to_ui',
      cluster: 'handler_started_request_not_sent',
      summary: '事件处理逻辑已开始执行，但未看到请求发出，疑似前置校验拦截、条件分支提前 return，或请求函数未真正调用。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_handler_trace',
        'ev_request_missing',
      ]),
      suggestions: [
        '检查 handler 中的前置校验分支',
        '检查请求函数调用条件是否被阻断',
        '检查异常是否在请求前被吞掉',
      ],
      isSymptomOnly: false,
      metadata: {},
    };
  }

  // =========================================================
  // Symptom / Assist Rules
  // =========================================================

  /**
   * R501: 页面显示 fallback 占位值
   * 说明：
   * - 纯症状规则，不直接作为最终根因
   */
  private hitFallbackDisplayedSymptom(
    context: DiagnosisContext,
  ): RuleFinding | null {
    const displayed = context.domValue ?? context.displayedValue;
    if (!this.isFallbackValue(displayed, context.fallbackTokens)) return null;

    return {
      ruleCode: 'R501',
      title: '页面显示 fallback 占位值',
      diagnosisLabel: '页面显示占位值',
      category: 'dom',
      severity: 'medium',
      confidence: 0.72,
      layer: 'ui',
      cluster: 'fallback_displayed',
      summary: '当前页面展示的是 fallback 占位值，这是异常表象，但不一定是根因。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_dom_snapshot',
        'ev_formatter_output_fallback',
      ]),
      suggestions: [
        '继续检查上游 API / state / render 链路',
        '定位 fallback 是在 render 生成还是 DOM 层兜底产生',
      ],
      isSymptomOnly: true,
      metadata: {
        displayed,
      },
    };
  }

  /**
   * R502: render 到 DOM 之间存在链路断点
   * 条件：
   * - renderTriggered = true
   * - renderCommitted = true 或 renderOutputValue 有值
   * - domUpdated = false
   */
  private hitRenderDomGapSymptom(
    context: DiagnosisContext,
  ): RuleFinding | null {
    const renderProduced =
      context.renderCommitted === true || !this.isNil(context.renderOutputValue);

    if (context.renderTriggered !== true) return null;
    if (!renderProduced) return null;
    if (context.domUpdated !== false) return null;

    return {
      ruleCode: 'R502',
      title: 'render 与 DOM 之间存在链路断点',
      diagnosisLabel: 'render 后 DOM 未接续',
      category: 'dom',
      severity: 'medium',
      confidence: 0.76,
      layer: 'state_to_render',
      cluster: 'render_dom_gap',
      summary: '渲染阶段已产出结果，但 DOM 未发生对应更新，问题可能出现在 commit、绑定或节点替换阶段。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_render_trace',
        'ev_dom_snapshot',
      ]),
      suggestions: [
        '检查 render commit 到 DOM 的链路',
        '检查节点 key/挂载点/条件分支切换',
      ],
      isSymptomOnly: true,
      metadata: {},
    };
  }

  /**
   * R503: 链路中断于 request 之前
   * 条件：
   * - mode = click_diagnosis
   * - clickDetected = true
   * - requestSent = false
   */
  private hitPreRequestGapSymptom(
    context: DiagnosisContext,
  ): RuleFinding | null {
    if (context.mode !== 'click_diagnosis') return null;
    if (context.clickDetected !== true) return null;
    if (context.requestSent !== false) return null;

    return {
      ruleCode: 'R503',
      title: '点击后链路中断于请求之前',
      diagnosisLabel: '请求前链路中断',
      category: 'interaction',
      severity: 'medium',
      confidence: 0.74,
      layer: 'request_to_ui',
      cluster: 'pre_request_gap',
      summary: '从点击到请求发送之间的链路出现中断，但尚不能仅凭该症状确定具体根因。',
      evidenceRefs: this.pickEvidenceRefs(context, [
        'ev_click_trace',
        'ev_request_missing',
      ]),
      suggestions: [
        '检查 handler 是否启动',
        '检查前置校验或节流防抖逻辑',
        '检查请求发送函数是否真正执行',
      ],
      isSymptomOnly: true,
      metadata: {},
    };
  }

  /**
   * R504: 证据不足但存在异常表象
   * 条件：
   * - 页面显示异常 / fallback
   * - 但上下游证据不足
   */
  private hitInsufficientEvidenceSymptom(
    context: DiagnosisContext,
  ): RuleFinding | null {
    const displayed = context.domValue ?? context.displayedValue;
    const abnormalDisplay =
      this.isFallbackValue(displayed, context.fallbackTokens) ||
      context.domVisible === false;

    if (!abnormalDisplay) return null;

    const evidenceCount = this.pickEvidenceRefs(context, []).length;
    const weakUpstream =
      context.responseSuccess !== true &&
      context.storeUpdated !== true &&
      context.renderTriggered !== true;

    if (!weakUpstream && evidenceCount >= 2) return null;

    return {
      ruleCode: 'R504',
      title: '证据不足但存在异常表象',
      diagnosisLabel: '证据不足',
      category: 'interaction',
      severity: 'low',
      confidence: 0.55,
      layer: 'ui',
      cluster: 'insufficient_evidence',
      summary: '当前页面存在异常表象，但关键链路证据不足，尚无法形成稳定根因判断。',
      evidenceRefs: this.pickEvidenceRefs(context, ['ev_dom_snapshot']),
      suggestions: [
        '补充接口响应快照',
        '补充状态更新轨迹',
        '补充 render trace 与 DOM snapshot',
      ],
      isSymptomOnly: true,
      metadata: {},
    };
  }

  // =========================================================
  // Helpers
  // =========================================================

  private isNil(value: unknown): boolean {
    return value === null || value === undefined;
  }

  private isFallbackValue(
    value: unknown,
    fallbackTokens?: string[],
  ): boolean {
    if (typeof value !== 'string') return false;
    const normalized = value.trim();
    return (fallbackTokens ?? []).includes(normalized);
  }

  private looseEqual(a: unknown, b: unknown): boolean {
    return String(a) === String(b);
  }

  private pickEvidenceRefs(
    context: DiagnosisContext,
    preferred: string[],
  ): string[] {
    const existing = new Set(context.evidenceRefs ?? []);
    const picked: string[] = [];

    for (const ref of preferred) {
      if (existing.has(ref)) picked.push(ref);
    }

    if (!picked.length) {
      return Array.from(existing).slice(0, 3);
    }

    return picked;
  }
}
```

# 4. 规则命中逻辑说明

下面把每条规则的设计意图说清楚，方便你后续继续扩。

---

## R101 接口返回成功但字段缺失
这是最“硬”的上游根因之一。

**证据特征：**
- request 成功
- 响应到了
- 目标字段没拿到

**适合解释：**
- 后端字段删了
- 接口 schema 变了
- 目标字段路径真的不存在

---

## R102 接口字段路径配置错误 / 绑定失效
这个和 R101 很像，但更偏前端绑定配置。

**区别：**
- `R101`：偏“返回里没有这个字段”
- `R102`：偏“前端取字段的路径错了”

在 MVP 阶段，这两条可能同时命中，属于正常现象。  
后面可以在 `RankingService` 中让：

- 后端实际没字段 → `R101` 更高
- schema 有变化 / binding config 更可疑 → `R102` 更高

---

## R103 请求成功但状态未更新
这是很多前端实际问题的高频断点。

**典型原因：**
- `dispatch`/`commit` 没执行
- reducer 被条件 return
- effect 成功但没写 store

---

## R104 状态已更新但 selector 未取到值
这是数据链路里很容易漏的断点。

**典型原因：**
- selector 路径写错
- selector 依赖的 state shape 改了
- memo 命中了旧结果

---

## R201 合法 falsy 值被误判为空
这是你前面反复在用的典型 MVP 根因规则。  
是非常适合作为 demo 的高价值规则。

**典型错误代码：**

```ts
if (!value) return '--';
```

而正确应该是：

```ts
if (value == null) return '--';
```

---

## R202 formatter 输出 fallback 占位值
比 R201 更泛化。  
只要 render 输入不空，但输出成了 fallback，就能命中。

---

## R203 render 输出异常
这是一个“泛异常规则”，用来兜住：
- 类型转换错误
- 小数格式错
- 单位拼接错
- map / enum transform 出错

它的置信度应低于 R201 / R202。

---

## R301 DOM 已渲染但被隐藏
这是 UI 层真实高频问题。

**常见原因：**
- `v-if`/条件渲染
- `display:none`
- 父层 overflow 裁剪
- 层级遮挡

---

## R302 DOM 未更新
说明 render 层已经有东西，但最终 UI 没接住。

**常见原因：**
- memo / shouldUpdate 阻断
- key 切换异常
- 绑定的 DOM 节点不是目标节点
- commit 没落到你观测的节点上

---

## R401 click 已发生但 handler 未启动
专门针对 click diagnosis 模式。

**常见原因：**
- 没绑事件
- 节点 disabled
- 冒泡被 stopPropagation
- 虚拟 DOM 替换导致 handler 丢了

---

## R402 handler 启动但 request 未发出
非常适合“点击后没反应”的问题。

**常见原因：**
- 参数校验失败后 return
- 前置 guard 拦截
- try/catch 吞掉异常
- 请求函数没调用到

---

## R501-R504 症状规则
这 4 条不要当最终根因，而应该：

- 补充 supporting causes / symptoms
- 给前端工作台一个“异常表象层”
- 在无强根因时也能返回可解释内容

这对 MVP 很重要，因为否则一旦没有命中强规则，结果页会显得“空”。

# 5. 当前版本的特点

这版实现有几个特点：

---

## 5.1 可直接接你现有骨架
它直接对接：

- `DiagnosisContext`
- `RuleFinding`
- `DiagnosisConclusionService`
- `RankingService`

不需要再改调用链。

---

## 5.2 允许多规则同时命中
这是有意设计的。  
因为真实诊断里通常是：

- 1 个 top cause
- 1~2 个 supporting causes
- 若干 symptoms

而不是“只能命中一条”。

---

## 5.3 先让 Rule Engine 偏“召回”
当前策略更偏 **recall-first**：

- 多命中
- 后面交给 ranking/conclusion 排序

这比在 Rule Engine 里写死互斥逻辑更适合 MVP 演进。

---

## 5.4 症状与根因分离
通过 `isSymptomOnly`，后面的 `DiagnosisConclusionService` 可以自然做：

- `topFinding`
- `supportingFindings`
- `symptomFindings`

这个结构对你的工作台展示特别重要。
