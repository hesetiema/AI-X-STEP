下面给你一版 **`network-timeline-builder.ts` 骨架**。  
目标很明确：

- 把 `NetworkInsight` 转成适合 **Popup / SidePanel / 工作台时间线** 展示的条目
- 文案尽量可读
- 保留最小技术引用
- 后续方便扩展 severity / tags / grouping

我会顺手补一个很小的 `TimelineItem` 类型扩展建议，和一个最小调用示例。

# 1. 先补一个类型建议

你前面 `types.ts` 里已经有：

```ts
export interface TimelineItem {
  id: string;
  type: 'network';
  occurredAt: number;
  title: string;
  subtitle?: string;
  severity?: 'info' | 'warning' | 'error';
  requestId: string;
  tags?: string[];

```

这已经够 MVP 了。  
如果你后面想更适合 UI，我建议可加两个可选字段，但现在不是必须：

```ts
// 可选增强，不是必须现在就加
meta?: Record<string, unknown>;

  method?: string;
  urlPattern?: string;
  status?: number;
  durationMs?: number;

```

先不改也行，下面骨架按你现有的 `TimelineItem` 来写。

# 2. `network-timeline-builder.ts`

```ts
// network-timeline-builder.ts

import type { NetworkInsight, TimelineItem } from './types';

export interface NetworkTimelineBuilder {
  build(insight: NetworkInsight, occurredAt: number): TimelineItem;
}

export class DefaultNetworkTimelineBuilder implements NetworkTimelineBuilder {
  build(insight: NetworkInsight, occurredAt: number): TimelineItem {
    const severity = this.resolveSeverity(insight);
    const title = this.buildTitle(insight);
    const subtitle = this.buildSubtitle(insight);
    const tags = this.buildTags(insight);

    return {
      id: this.buildTimelineId(insight, occurredAt),
      type: 'network',
      occurredAt,
      title,
      subtitle,
      severity,
      requestId: insight.requestId,
      tags: tags.length > 0 ? tags : undefined,
    };
  }

  private buildTimelineId(insight: NetworkInsight, occurredAt: number): string {
    return `network-${insight.requestId}-${occurredAt}`;
  }

  private buildTitle(insight: NetworkInsight): string {
    // MVP 先直接使用业务动作标题
    // 例如：
    // - 查询订单列表
    // - 提交创建
    // - 查询详情
    return insight.actionLabel;
  }

  private buildSubtitle(insight: NetworkInsight): string | undefined {
    const parts: string[] = [];

    if (insight.responseText) {
      parts.push(insight.responseText);
    }

    const debugHint = insight.debugHints?.[0];
    if (debugHint) {
      parts.push(debugHint);
    }

    return parts.length > 0 ? parts.join(' · ') : undefined;
  }

  private resolveSeverity(
    insight: NetworkInsight,
  ): 'info' | 'warning' | 'error' {
    switch (insight.resultCategory) {
      case 'failed':
      case 'timeout':
        return 'error';
      case 'empty':
      case 'invalid_data':
        return 'warning';
      case 'success':
      case 'unknown':
      default:
        return 'info';
    }
  }

  private buildTags(insight: NetworkInsight): string[] {
    const tags: string[] = [];

    if (insight.module) {
      tags.push(insight.module);
    }

    if (insight.resource) {
      tags.push(insight.resource);
    }

    tags.push(insight.resultCategory);

    const status = insight.rawRef.status;
    if (typeof status === 'number') {
      tags.push(`http:${status}`);
    }

    if (insight.confidence) {
      tags.push(`confidence:${insight.confidence}`);
    }

    return tags;
  }
}
```

# 3. 最小使用示例

```ts
import { DefaultNetworkTimelineBuilder } from './network-timeline-builder';
import type { NetworkInsight } from './types';

const builder = new DefaultNetworkTimelineBuilder();

const insight: NetworkInsight = {
  requestId: 'req-001',
  actionKey: 'queryOrders',
  actionLabel: '查询订单列表',
  module: 'order-list',
  resource: 'order',
  confidence: 'high',
  resultCategory: 'empty',
  requestText: '查询订单列表（筛选条件: page, status）',
  responseText: '接口成功返回，但结果为空',
  debugHints: [
    '页面可能展示为空列表或空态',
    '请确认筛选条件是否过严',
  ],
  rawRef: {
    method: 'GET',
    urlPattern: '/api/orders',
    status: 200,
    durationMs: 420,
  },
};

const item = builder.build(insight, Date.now());

console.log(item);
```

输出大致会是：

```ts
{
  id: 'network-req-001-1718611111111',
  type: 'network',
  occurredAt: 1718611111111,
  title: '查询订单列表',
  subtitle: '接口成功返回，但结果为空 · 页面可能展示为空列表或空态',
  severity: 'warning',
  requestId: 'req-001',
  tags: ['order-list', 'order', 'empty', 'http:200', 'confidence:high']
}
```

# 4. 这版 builder 的职责边界

这点我建议你后面文档里也明确一下。

## 它负责
- 把 `NetworkInsight` 变成 **UI 可展示条目**
- 给出：
  - `title`
  - `subtitle`
  - `severity`
  - `tags`

## 它不负责
- 网络事件分析
- 业务映射判断
- 响应结构解析
- 最终诊断结论

也就是说：

> `network-timeline-builder` 是展示适配层，不是分析层。

# 5. 我建议的后续增强点

这版骨架能跑，但后面很可能会继续补这几类能力。

---

## 5.1 针对不同结果类别输出更自然标题
比如：

### 当前
- `title = 查询订单列表`

### 后续可以增强成
- 成功：`查询订单列表`
- 空结果：`查询订单列表（空结果）`
- 超时：`查询订单列表超时`
- 失败：`查询订单列表失败`

如果你想更强调时间线可读性，这个很值得加。

---

## 5.2 支持 UI 分组字段
比如给 timeline item 加：

```ts
group?: 'network' | 'interaction' | 'error' | 'ui_state';
```

虽然现在 `type: 'network'` 已经够了，但后面统一时间线时会更舒服。

---

## 5.3 支持二级详情引用
例如：
- hover 展示 method / path / duration
- 点击跳到 evidence drawer

可以考虑后续给 `TimelineItem` 增加：

```ts
meta?: {
  method: string;
  urlPattern: string;
  status?: number;
  durationMs?: number;
}
```

---

## 5.4 慢请求标记
现在只是 `failed/timeout => error`。  
后面可以把：

- duration > 2000ms
- duration > 5000ms

映射成：
- `warning`
- `error`

即使它最终 200 成功，也能提示“慢请求”。

# 6. 如果你想更适合 SidePanel，我建议一个增强版标题策略

如果你愿意稍微多一步，可以把 `buildTitle` 改成这样：

```ts
private buildTitle(insight: NetworkInsight): string {
  switch (insight.resultCategory) {
    case 'failed':
      return `${insight.actionLabel}失败`;
    case 'timeout':
      return `${insight.actionLabel}超时`;
    case 'empty':
      return `${insight.actionLabel}（空结果）`;
    case 'invalid_data':
      return `${insight.actionLabel}（返回异常）`;
    default:
      return insight.actionLabel;
  }
}
```

这样 SidePanel 时间线会更一眼可读。  
不过如果你想保持“标题稳定 + 细节在 subtitle”，当前版本也没问题。

# 7. 推荐和前面模块的串联方式

现在你已经有这几个模块：

- `network-sanitizer.ts`
- `network-normalizer.ts`
- `mapping-registry.ts`
- `default-mappings.ts`
- `network-insight-transformer.ts`
- `network-timeline-builder.ts`

可以串成这条链：

```ts
rawEvent
  -> sanitizer.sanitize(rawEvent)
  -> normalizer.normalize(sanitizedEvent)
  -> transformer.transform(context)
  -> timelineBuilder.build(insight, occurredAt)
```

最终 session 中建议同时保留：

1. `normalized network event`
2. `network insight`
3. `timeline item`

这样后面 UI 和 diagnosis 后端都好接。
