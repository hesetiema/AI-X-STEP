# 2. 统一规则判断约定

为了后续 rule spec 稳定，建议这批规则先统一采用下面思路：

- **R1xx**：接口 / state / selector 上游问题
- **R2xx**：render/formatter/selector 中游问题
- **R3xx**：DOM 层问题
- **R4xx**：交互 / 请求前链路问题
- **R5xx**：UI 症状 / 展示症状

并保持：

- `isSymptomOnly = true` 只给症状规则
- `layer` 用于 ranking / domino
- `cluster` 用于聚类去重
- `confidence` 先固定，后面再调

## 3.1 `r102-api-null-value.rule.ts`

```ts
import { Injectable } from '@nestjs/common';

import { DiagnosisContext } from '../models/diagnosis-context.model';
import { RuleFinding } from '../models/rule-finding.model';
import { DiagnosisRule } from './interfaces/diagnosis-rule.interface';

@Injectable()
export class R102ApiNullValueRule implements DiagnosisRule {
  code = 'R102';

  evaluate(context: DiagnosisContext): RuleFinding | null {
    if (!context.responseSuccess) return null;
    if (!context.apiFieldExists) return null;

    const isNullish = context.apiValue === null || context.apiValue === undefined;
    if (!isNullish) return null;

    return {
      ruleCode: 'R102',
      title: '接口字段值为空',
      diagnosisLabel: '接口字段值为空',
      category: 'data_source',
      severity: 'high',
      confidence: 0.92,
      layer: 'api',
      cluster: 'api_null_value',
      summary: `接口字段 ${context.apiFieldPath ?? '目标字段'} 存在，但值为空。`,
      evidenceRefs: ['ev_api_null_value'],
      suggestions: ['检查接口数据源是否返回空值', '检查后端字段赋值逻辑'],
      isSymptomOnly: false,
    };
  }
}
```

## 3.4 `r201-falsy-swallowed.rule.ts`

```ts
import { Injectable } from '@nestjs/common';

import { DiagnosisContext } from '../models/diagnosis-context.model';
import { RuleFinding } from '../models/rule-finding.model';
import { DiagnosisRule } from './interfaces/diagnosis-rule.interface';

@Injectable()
export class R201FalsySwallowedRule implements DiagnosisRule {
  code = 'R201';

  evaluate(context: DiagnosisContext): RuleFinding | null {
    if (!context.selectorRan) return null;

    const value = context.selectorValue;
    const isFalsyButMeaningful =
      value === 0 || value === false || value === '';

    if (!isFalsyButMeaningful) return null;
    if (context.renderInputValue === value) return null;

    return {
      ruleCode: 'R201',
      title: '有效 falsy 值被吞掉',
      diagnosisLabel: '有效 falsy 值被吞掉',
      category: 'render_logic',
      severity: 'high',
      confidence: 0.89,
      layer: 'render',
      cluster: 'falsy_swallowed',
      summary: 'selector 已产出有效 falsy 值，但渲染输入未保留该值，疑似被 `||` 等逻辑吞掉。',
      evidenceRefs: ['ev_falsy_swallowed'],
      suggestions: ['检查 `||` 默认值逻辑', '优先改用 `??` 处理空值'],
      isSymptomOnly: false,
    };
  }
}
```

- 固定 `createContext`

- explanation

## 11.2 断言以稳定字段为主
优先断言：

- `ruleCode`
- `layer`
- `cluster`
- `isSymptomOnly`

---

## 11.3 negative case 至少两个
每个规则尽量至少测：

- 一个命中 case
- 一个前置条件不满足
- 一个关键差异条件不满足

这样覆盖就比较扎实。
