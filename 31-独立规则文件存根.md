下面直接给你 **rules 独立文件版 stub**，目标是把你现在的 `rule-engine.service.ts` 从“内嵌规则 if/else”升级成：

- `DiagnosisRule` 接口
- `Rule Registry`
- `R101 ~ R504` 独立规则类
- `RuleEngineService` 改为遍历 registry 执行

这版还是按 **NestJS 可直接落地** 来写。

import { Injectable } from '@nestjs/common';
import { DiagnosisRule } from '../interfaces/diagnosis-rule.interface';
import { DiagnosisContext } from '../../models/diagnosis-context.model';
import { RuleFinding } from '../../models/rule-finding.model';
import { isFallbackValue } from '../../utils/diagnosis-value.utils';

@Injectable()
export class R201FalsyValueSwallowedRule implements DiagnosisRule {
  readonly code = 'R201';

  evaluate(context: DiagnosisContext): RuleFinding | null {
    const hasValidFalsyUpstream =
      context.apiValue === 0 ||
      context.apiValue === false ||
      context.storeValue === 0 ||
      context.storeValue === false ||
      context.selectorValue === 0 ||
      context.selectorValue === false ||
      context.renderInputValue === 0 ||
      context.renderInputValue === false;

    if (
      hasValidFalsyUpstream &&
      (context.formatterOutputIsFallback ||
        isFallbackValue(context.renderOutputValue, context.fallbackTokens) ||
        isFallbackValue(context.domValue, context.fallbackTokens))
    ) {
      return {
        ruleCode: this.code,
        title: '合法 falsy 值被误判为空',
        diagnosisLabel: '合法值被判空吞掉',
        category: 'render_transform',
        severity: 'high',
        confidence: 0.96,
        layer: 'render',
        cluster: 'formatter_falsy_swallowed',
        summary: '合法值被错误当成空值处理。',
        evidenceRefs: ['ev_api_amount_0', 'ev_render_trace'],
        suggestions: ['将 if (!value) 改为 value == null'],
        isSymptomOnly: false,
      };
    }

    return null;
  }
}

## `src/modules/diagnosis/domain/services/rule-engine.service.ts`

```ts
import { Injectable } from '@nestjs/common';
import { DiagnosisContext } from '../models/diagnosis-context.model';
import { RuleFinding } from '../models/rule-finding.model';
import { DiagnosisRuleRegistry } from '../rules/registry/diagnosis-rule.registry';

@Injectable()
export class RuleEngineService {
  constructor(private readonly diagnosisRuleRegistry: DiagnosisRuleRegistry) {}

  run(context: DiagnosisContext): RuleFinding[] {
    const findings = this.diagnosisRuleRegistry
      .getRules()
      .map((rule) => rule.evaluate(context))
      .filter((item): item is RuleFinding => !!item);

    return this.dedupeFindings(findings);
  }

  private dedupeFindings(findings: RuleFinding[]): RuleFinding[] {
    const map = new Map<string, RuleFinding>();

    for (const item of findings) {
      const key = `${item.ruleCode}::${item.layer}::${item.cluster ?? ''}`;
      if (!map.has(key)) {
        map.set(key, item);
      }
    }

    return [...map.values()];
  }
}
```

# 12. 这版结构的好处

这次改完后，你的 rule engine 已经从“集中 if/else”进入 **可扩展规则架构** 了，后面可以直接做：

---

## 12.1 单条规则单测更容易写
比如：

- `r201-falsy-value-swallowed.rule.spec.ts`
- `r401-click-handler-not-started.rule.spec.ts`

每条规则可以独立测，不依赖完整 engine。

---

## 12.2 registry 可按阶段配置
后面你可以很容易扩成：

```ts
getRules(mode?: DiagnosisMode): DiagnosisRule[]
```

或者：

```ts
getRulesByScenario('inspect' | 'click')
```

这样 click 场景不跑 inspect-only 规则。

---

## 12.3 后续支持元数据更自然
如果后面想在规则里加：

- `version`
- `owner`
- `enabled`
- `tags`
- `priority`

可以直接扩接口：

```ts
export interface DiagnosisRule {
  readonly code: string;
  readonly meta?: {...};
  evaluate(...): RuleFinding | null;
}
```
